# SQL KPI Analysis Project

## Overview
This project was completed during my SQL coaching program.
The objective was to analyze a churn dataset and derive
25–30 business KPIs using SQL.

The project focuses on understanding customer behavior,
credit scores, activity status, and churn patterns.

## Dataset Information
- Total records: 10,000
- Total columns: 14
- Data quality: Mostly clean, minor issues in customer surnames
- Key columns include:
  - CreditScore
  - Geography
  - Gender
  - Age
  - Tenure
  - Balance
  - HasCrCard
  - IsActiveMember
  - Exited
  - EstimatedSalary

## Tools Used
- SQL (MySQL)
- Excel (dataset)

## KPIs Derived
Some of the KPIs include:
- Total and unique customers
- Credit score statistics (min, max, average)
- Age and tenure analysis
- Active vs inactive customers
- Credit card ownership analysis
- Gender-based insights
- Geography-based insights
- Churned vs non-churned customers
- Estimated salary analysis

## Repository Contents
- `Churn_modelling` – Churn dataset
- `churn_queries` – SQL queries for all KPIs
- `Report` – KPI report and summary

## Notes
This project focuses on SQL-based KPI analysis.
No machine learning or predictive modeling was performed.
